# EchoStream

## 首选项

- 语言：默认输出请使用简体中文，包括代码注释、消息和文档。
- 测试：除非调试问题，开发过程中不要给出测试代码（后面会有专门的环节补全测试用例）

## 重要原则

- 轻量
  - 最小化外部依赖
  - 最精简的代码逻辑，少了不行，多了不要
- 简单
  - 大道至简，每个接口设计和模块封装，使用起来都应该足够简单、方便、符合直觉和常识
  - 让开发体验更好，使用效率更高，维护更简单，降低他人的使用心智和复杂度
- 性能
  - 在不显著增加复杂度的前提下
  - 尽可能追求更高的性能、更小的内存占用和运行开销
- 清晰
  - 文档要通俗易懂，言简意赅，切莫长篇大论
  - 给出核心使用场景的示例代码，而不是每个参数细节都罗列展示
  - 大家的精力都很宝贵，把最关键的信息和操作路径给出来，让人快速上手


## 注意事项

- 在设计子模块的使用场景和 API 示例时，不要和主模块的 EchoServer / EchoClient 产生关联（后面会手动添加调用方法）
- 变量方法命名要符合直觉和常识，优先采用业界常用命名习惯，使用起来要足够直观，没有歧义
